# Examples

### Fractals
- **Sierpinski Triangle** and **Koch Snowflake**: generated by consecutive rule applications starting from small initial host graphs
- Running time increases exponentially with the number of iterations; yComp's layout algorithm may also need time
- Scripts use debug mode; press `r` to run to completion

### Busy Beaver (Turing Machine)
- A five-state busy beaver Turing machine that writes 1,471 bars and terminates; demonstrates Turing completeness
- Model: tape as a chain of `TapePosition` nodes linked by `right` edges; cell value as a reflexive `value` edge (subtypes: `zero`, `one`, `empty`); `RWHead` edge from a `State` node to the current `TapePosition`; `WriteValue` nodes (subtypes: `WriteZero`, `WriteOne`, `WriteEmpty`) with `moveLeft`/`moveRight`/`dontMove` edges to the next state
- 11 rules implementing a generic Turing machine: 3 read rules, 3 write rules, 3 move rules, 2 tape-extension rules (with negative patterns checking that no left/right neighbour exists yet)
- Execution sequence: `((readOneRule | readEmptyRule) & (writeOneRule | writeEmptyRule) & (ensureMoveLeftValidRule | ensureMoveRightValidRule) & (moveLeftRule | moveRightRule))*`
- After 32 warm-up steps: `custom graph analyze` + `custom actions gen_searchplan` on all rules regenerates better search plans — reducing execution from ~90 sec to ~8.5 sec (by starting lookup at the unique `RWHead` edge instead of the many `right` edges)
