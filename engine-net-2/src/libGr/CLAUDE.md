# libGr

Core graph abstraction layer - defines the fundamental graph interfaces of GrGen.NET.

## Purpose

Provides the contract between Java-generated code and the C# backend runtime. All graph operations, pattern matching interfaces, and (meta) type systems are defined here. Also the sequence parser and interpreter are contained here. This is the largest and most central library in GrGen.NET.

The interfaces defined here are implemented in different places: some by the lgspBackend (e.g. graph and action runtime), some by the code generated by the Java frontend (e.g. generated action and model classes), and some by other libraries (e.g. `IPersistenceProvider` by `libGrPersistenceProviderSQLite`).

The library also contains quite some helper code on top of the basic interfaces (central parts of the language runtime, with implementations of e.g. the reachability functions from the global namespace, or the import/export code).

## Key Interfaces

### IBackend

Main entry point for backend implementations:
- `CreateGraph()` / `CreateNamedGraph()` - Graph creation
- `CreateFromSpec()` - Create graph from .grg file
- `ProcessSpecification()` - Compile .grg to assemblies
- `LoadActions()` - Load action assemblies

### IGraph / INamedGraph

Graph manipulation:
- Node/edge creation, deletion, retyping
- Attribute access
- Graph traversal (incident, adjacent, reachable)
- Cloning and merging
- Change events (`OnNodeAdded`, `OnRemovingNode`, `OnChangingNodeAttribute`, `OnRetypingNode`, etc.) — consumed by graph visualization, persistence providers, and other observers

### IPersistenceProvider

Interface for persisting a named graph to an external repository (e.g. a database). Extends `IPersistenceProviderStatistics` and `IPersistenceProviderTransactionManager`.

### IAction

Pattern matching and rewriting:
- `Match()` - Find pattern matches
- `Modify()` - Apply rewrite
- `Apply()` - Match and rewrite in one step

### ProcessSpecFlags

Compilation options:
- `UseNoExistingFiles`, `UseJavaGeneratedFiles`, `UseAllGeneratedFiles`
- `KeepGeneratedFiles`, `CompileWithDebug`
- `NoEvents`, `NoDebugEvents`
- `LazyNIC`, `Noinline`, `Profile`
